# 🎓 University Notice System Project Roadmap

이 프로젝트는 대학교 공지사항 및 다양한 소스의 정보를 통합하여, 사용자가 설정한 키워드에 맞는 알림을 제공하는 시스템입니다.

---

## 🛠 Phase 1: 데이터 소스 및 모델 표준화 (진행 중)
- [x] **공지사항 데이터 모델(`Notice`) 정의**: 제목, 내용, 카테고리, 출처, 날짜 필드 표준화
- [x] **Jsoup 기반 크롤링 로직 검증 (`Crawl.java`)**: 광운대 사이트 대상 제목/날짜/링크 추출 성공
- [ ] **데이터 소스 통합**: `Crawl.java`의 로직을 `PortalNoticeSource` 등 실제 소스 클래스로 이식
- [ ] **중복 데이터 소스 정리**: 하드코딩된 샘플 데이터 소스(`Department...`, `Activity...`) 정리 및 기능 재정의

## 🔍 Phase 2: 키워드 필터링 및 사용자 관리
- [ ] **사용자별 키워드 설정 기능**: `User` 클래스에 관심 키워드 리스트 관리 기능 추가
- [ ] **키워드 필터링 엔진 고도화**: `NoticeFilter`를 통해 제목/내용에서 키워드 포함 여부 판별
- [ ] **알림 로직 구현**: 새로운 공지가 올라왔을 때 콘솔 또는 시스템 알림으로 표시

## 💾 Phase 3: 데이터 지속성 (Persistence)
- [ ] **DB 구조 설계**: 공지사항의 중복 저장을 방지하고 효율적으로 관리하기 위한 DB 스키마 설계
- [ ] **저장 로직 구현**: 처음 발견된 공지만 DB에 저장하는 `Duplicate Check` 로직 추가
- [ ] **설정 저장**: 사용자의 키워드 설정을 프로그램 종료 후에도 유지되도록 저장 (JSON 또는 SQLite)

## 🖥 Phase 4: 사용자 인터페이스 (GUI)
- [ ] **메인 대시보드 구현**: 관심 키워드별 공지사항을 한눈에 볼 수 있는 게시판 형태의 UI
- [ ] **설정 화면**: 키워드 추가/삭제 및 알림 설정을 할 수 있는 관리 화면
- [ ] **시스템 트레이 아이콘**: 백그라운드에서 돌면서 새 공지 발생 시 팝업 알림 제공

## 🚀 Phase 5: 소스 확장 및 고도화
- [ ] **다양한 소스 추가**: 인스타그램(공식 계정), 에브리타임(스크래핑 가부 확인), 타 학과 사이트 등 추가
- [ ] **상세 내용 크롤링**: 현재 링크만 가져오는 방식에서, 상세 페이지 내부의 텍스트까지 분석하도록 고도화
- [ ] **반자동 스케줄링**: 특정 시간 간격으로 자동으로 크롤링이 수행되도록 타이머 기능 추가

---

> **Tip**: 각 단계가 완료될 때마다 이 로드맵의 `[ ]`를 `[x]`로 업데이트하며 진행합니다.
